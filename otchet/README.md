# /otchet — живой реестр гипотез

Публичная страница: https://constructor.lead-generator.ru/otchet/ (noindex).
Назначение: презентация прогресса проекта — что пробовали, что проверили, какая оценка, куда идём дальше.

## Устройство

- `index.html` — рендерер (статичный, без сборки). Меняется редко.
- `data.json` — весь контент: гипотезы, векторы, таймлайн, KPI. **Обновляется пачками** — это основной файл системы.

## Цикл гипотезы

```
идея → status:"open" → status:"testing" → проверка живым прогоном
     → Claude пишет оценку (score 0–10 + impact) → status:"worked|partial|failed|abandoned"
     → из результата выводится next_vector (куда дальше)
```

## Поля гипотезы в `data.json`

| Поле | Что писать |
|---|---|
| `id` | `H48`, `H49`, … — следующий свободный номер |
| `title` | короткое имя гипотезы |
| `description` | что меняем и какого эффекта ждём (формулировка гипотезы) |
| `status` | `open` → `testing` → `worked` / `partial` / `failed` / `abandoned` |
| `score` | 0–10 после проверки (`null` пока не проверено). 0–3 не дало/вредно · 4–6 умеренно · 7–8 сильно, в проде · 9–10 определяющее |
| `evidence` | **факты**: коммиты, замеры, скриншоты, даты — без интерпретации |
| `impact` | **оценка Claude**: что реально изменилось, простыми словами, 1–3 предложения |
| `next_vector` | куда двигаться дальше исходя из результата (`null` если направление закрыто) |
| `category` | область: генерация / дизайн / верификация / LLM-провайдеры / рантайм / автономность / продукт / инфра / безопасность / процесс |
| `period` | даты жизни гипотезы, например `"01.07 — 07.07"` |

Правила стиля записей — раздел «Стиль изложения» в корневом `CLAUDE.md`: развёрнуто без воды, простыми словами, цифры и коммиты не терять.

## Как обновить (для Claude-сессии)

1. Отредактировать `otchet/data.json`: добавить/обновить гипотезы, поднять `meta.updated` (`YYYY-MM-DD`).
2. При необходимости обновить `vectors` (активные направления) — каждый вектор ссылается на гипотезы по `hyp_ids`.
3. Локальная проверка: `python -m http.server` в этой папке, открыть, убедиться что JSON валиден и рендер живой.
4. Коммит + push в `main`.
5. Деплой на VPS: см. раздел деплоя в корневом `CLAUDE.md` (страница отдаётся nginx-ом с VPS).

## Чего НЕ делать

- Не переписывать `index.html` под каждую правку данных — контент живёт в `data.json`.
- Не удалять провалившиеся гипотезы — они зачёркиваются на странице автоматически (status `failed`/`abandoned`); провал — такой же результат, как успех.
- Не ставить оценку без живой проверки — сначала прогон, потом score.
