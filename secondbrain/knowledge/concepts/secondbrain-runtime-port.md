# SecondBrain Runtime Port

Porting the SecondBrain runtime components (scripts, hooks, templates) from CorporateMessenger to the Omnia project, adapting them to the new project structure and configuration.

## Key Points

- Bulk-copied `scripts/`, `hooks/`, and `templates/` directories.
- Adapted `sync_project_docs.py` for Omnia's `docs/` and `agents/` paths.
- Modified `update_context.py` for `omnia-mvp/` path splitting.
- Adjusted `hooks/post-tool-use.py` for Omnia's `CONTEXT_TRIGGERS`.
- Created Omnia-specific README, AGENTS, project-context, index, log.
- Configured `.claude/settings.json` with five essential hooks.
- Added `.gitignore` entries for state files.

## Details

The SecondBrain runtime, previously used in CorporateMessenger, was ported to the Omnia project. This involved a bulk copy of critical directories: `scripts/` (16 files), `hooks/` (5 files), and `templates/`. This foundational transfer brings the core intelligence and automation capabilities to Omnia.

Significant adaptation was required to align the ported components with Omnia's project structure. `sync_project_docs.py` was modified to correctly reference `docs/`, `agents/`, and `apps/*/CLAUDE.md` for documentation ingestion. `update_context.py` was adjusted to handle path splitting specific to the `omnia-mvp/` directory structure.

The `hooks/post-tool-use.py` script was updated to recognize `CONTEXT_TRIGGERS` relevant to Omnia's architecture. Additionally, Omnia-specific documentation such as README, AGENTS, project-context, index, and log files were created to provide relevant context for the AI.

The `.claude/settings.json` file was configured to include five essential hooks: SessionStart, SessionEnd, PreCompact, PostToolUse, and Stop, enabling comprehensive control over AI interactions. Finally, `.gitignore` entries were added to manage state files generated by the SecondBrain runtime.
