### Real-time «строится на глазах»: resumable stream + фотки в рамки + code-stream (deployed @f94bf2f)

**Запрос владельца:** превью должно гипнотизировать — сайт строится на глазах, фотки **въезжают

**3 фазы, каждая отдельным коммитом + деплоем на прод (`full`/omnia-prod-*), проверена:**

- **Phase 1 — resumable stream** (`0e734e2`). Root cause F5-бага: WS открывался ТОЛЬКО в
  `submit()`, на reload reconnect-логики не было, pub/sub эфемерен → превью мёрзло, бэкенд стримил
  в мёртвый сокет. Fix: API буферит кумулятивный контент в Redis на каждую дельту
  (`omnia:stream:<msg>` + `omnia:active:<project>`, TTL 600s, чистится в `finally`), каждый
  `llm.chunk` несёт монотонный `seq`; на (ре)коннекте WS шлёт `stream.sync` ДО подписки на live;
  клиент при gap шлёт `resync`. Фронт (`usePromptStream`) реконнектит на mount при висящем
  in-flight ассистенте, дедупит по seq, авто-reconnect с backoff. **Проверено детерминир. скриптом
  в проде: 5/5 PASS** (stream.sync реплеит буфер, resync, cleanup) — без LLM-затрат.
- **Phase 2 — фотки въезжают в рамки** (`0948a43`). `resolve_images` теперь `asyncio.as_completed`
  (было `gather`) + callback `on_image(preview_idx,url)` на КАЖДУЮ готовую картинку; `preview_idx`
  = позиция среди `img[data-omnia-gen]` в index.html. messages.py публикует `image.resolved`
  (флаг `use_live_image_events`, default on). Фронт: client-cache `["stream-images",pid,msg]` →
  `StreamingPreviewFrame` шлёт `omnia:image` в iframe → bootstrap shimmer-рамит пустые imgs и
  «оседает» фото (blur+scale→резкость), переживает morphdom-патчи. **Проверено локально: 6/6 PASS**
  (индексы, группы шарят картинку, контракт resolve не сломан).
- **Phase 3 — live code-stream** (`f94bf2f`). `StreamingCodeView` парсит partial-файлы из стрима

Решения: элемент + тумблер «весь сайт»; Save = одна версия. Инструменты: sequential-thinking + context7

**Движок (паритет live↔persist):** managed `<style id="omnia-overrides">` (правила `!important`, ключ =
CSS-селектор инспектора, последним в head) + `<link data-omnia-font>`. Live → инспектор в ОТДЕЛЬНЫЙ
`omnia-overrides-live` (не трогает committed, чтобы новая сессия не стёрла сохранённое); Save → бэкенд
МЕРЖИТ в committed (парсит старый блок → кумулятивно). `carry_over_overrides` в messages.py перед commit
→ переживает AI-регены. Гарды (palette/contrast) не трогают (правят только первый :root + body).

**Бэкенд:** `services/overrides.py` (pure, merge+carry-over), `POST …/style-patch` (зеркало rollback;
skip-guards но reject banned-hex/unknown-font→400; font href с сервера), `GET /api/fonts`
(`services/fonts.py` — один источник для пикера и валидатора). Инспектор `omnia-inspector.js` ×2
байт-в-байт (drift-тест): styleMode + omnia:style:set/font:link/reset + extended pick (computed color/font).

**Фронт:** `store/styleEdit.ts`, `lib/api/style.ts`, `StylePanel.tsx` (Текст/Фон/Граница + color input +
EyeDropper в РОДИТЕЛЬСКОМ окне feature-detect + тумблер токенов + список шрифтов своим шрифтом + Save).
`PreviewFrame` Pipette-тумблер (взаимоисключающ с select-mode). WS закрыт в простое → панель сама
`invalidateQueries(["snapshots"])`.

**E2E на проде (playwright, проект sushi-restoran-023d66, 0 console errors):** style-тумблер → клик h1 →
панель с computed цветом/шрифтом → цвет #00ff88 применился ВЖИВУЮ (h1 color=rgb(0,255,136)) → шрифт
