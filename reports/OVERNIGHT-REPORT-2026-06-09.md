## Что сделано за ночь (по порядку)

### P0 — Инфраструктура: превью больше не «засыпает» под зрителем
Раньше открытые превью-аппы могли «уснуть» прямо когда на них смотрят (система их глушила, считая неактивными) — из-за этого нельзя было нормально тестировать ни дизайн, ни превью. Починили:
- Пока на превью смотрят — система это видит (по сетевому трафику) и не усыпляет.
- Память контейнера поднята с 2 ГБ до **4 ГБ** — тяжёлые приложения больше не падают от нехватки памяти.
- Контейнер сам поднимается после сбоя.
**Итог:** превью стабильно живёт, тяжёлые аппы не падают. (коммит c7fc2fe)

### P1 — Главная цель MVP: вход без трения → чат сам всё выясняет → собирает приложение
Это самый большой блок ночи. Теперь путь пользователя такой, как вы хотели:
- Нажал «Новый проект» → **сразу попадаешь в рабочее пространство** (никакого выбора шаблона, никакого квиза).
- **Чат сам, по одному простому вопросу**, выясняет, что за приложение нужно (цель, аудитория, ключевая фича…), подстраивается под ответы.
- **Сам подбирает технологию**: простой лендинг или полноценное приложение с базой данных — решает по вашему описанию.
- **Сам собирает приложение** и поднимает его (без кнопки «Запустить»).
- **Удаление проекта**: на карточке проекта меню (⋮) → «Удалить» → ввести имя для подтверждения → проект и вся его инфраструктура (контейнер, база, домен) сносятся. Чужой проект удалить нельзя. Данные базы не стираются физически сразу — есть окно на восстановление.

Попутно починили **3 серьёзных бага**, из-за которых полноценные приложения раньше вообще не собирались корректно (падали при сборке или рендере). Теперь собираются — проверено на реальном магазине косметики (каталог, регистрация, корзина — всё работает).
**Итог: P1 закрыт полностью.** (коммиты 56ac129, 202b7cc, 52e91c2, c95ef13 и др.)

### P2 — Ошибки приложения показываются красивыми карточками в чате
Если созданное приложение ломается, пользователь теперь это видит прямо в чате аккуратной красной карточкой (тип ошибки, файл, детали) + кнопка **«Починить»**, которая сама запускает исправление. Покрыты все виды ошибок:
- ошибки сборки/компиляции,
- ошибки базы данных,
- серверные ошибки (страница падает с 500 при открытии),
- ошибки в браузере (JS-ошибки на превью).
**Итог: P2 закрыт полностью.** (коммиты ede1c57, e15387b, 3dcc42b)

### P3 — Ручные правки и «кликер» в полноценных приложениях
- **Ручные правки** (клик по элементу → смена цвета/фона/удаление) теперь **сохраняются и в полноценных приложениях** (раньше молча падали). Проверено вживую.
- **Кликер** (тык в элемент → ИИ точечно правит) — разобрались: он на самом деле подключён, «поломка» — это точность дешёвой модели, а не сам механизм. Оставлено как задача на улучшение качества.
**Итог: ручные правки работают.** (коммиты 46a4ac2, 113405b)

## На что посмотреть утром
1. **Главное:** зайдите как новый пользователь → «Новый проект» → только название → попадёте сразу в чат, который сам расспросит и соберёт приложение. Это и есть закрытый MVP-путь.
2. **Откаты** на случай проблем (каждая фича за флагом/коммитом): прогрессивная дискавери `USE_PROGRESSIVE_DISCOVERY=false`; авто-стек `USE_AUTO_STACK_ROUTING=false`; карточки ошибок `USE_ERROR_CARDS=false`.
3. **Тестовый мусор на проде:** за ночь насоздавалось несколько throwaway-проектов (avtostek-*, magazin-pomady, render-magazin-*, кофейни, throwaway-аккаунты) — безвредны, можно снести через удаление проектов (оно теперь работает).
4. **Ограничения, отмеченные на будущее:** на полных аппах смена шрифта на совсем новый webfont пока не подтягивает начертание (цвет/фон работают); миниатюра иногда ловит ранний кадр загрузки.
