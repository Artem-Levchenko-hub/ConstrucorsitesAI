# 11. Свободная генерация вёрстки + приёмочный контроль (уход от трафаретов)

> **Статус:** план, утверждён владельцем 2026-05-30. Реализация по спринтам ниже.
> **Зоны:** основное — Agent B (`apps/api/`); vision в gateway — Agent C (`apps/llm-gateway/`); UI-индикаторы качества (опционально) — Agent A (`apps/web/`).

## Контекст и решение

Текущий движок генерации статических сайтов работает в режиме **«каталог/IR»**: на сильных моделях (premium tier) LLM выдаёт только `PageIR` JSON, а финальный HTML собирает детерминированный Jinja-рендерер из **16 фиксированных трафаретов** (`apps/api/src/omnia_api/sections/`). Это даёт надёжность (валидный, адаптивный сайт без «белого экрана»), но ценой шаблонности: два сайта с одинаковым выбором секций **байт-в-байт идентичны по вёрстке**, палитра берётся всегда первая из 54, шрифты — свободный текст со скатыванием в дефолт, а аудит проверяет только дисциплину и не отличает «генерик» от «красиво».

**Решение владельца:** отказаться от трафаретов как от основного пути. Сильные модели рисуют вёрстку **свободно**, опираясь на нашу методичку-промпт + дизайн-токены + kit-библиотеку. Надёжность сохраняем не фиксацией результата, а **приёмочным контролем** (рендер → скриншот → проверка → авто-починка до показа). Трафареты остаются **аварийным fallback**.

### Принцип

> **Свобода в композиции — жёсткость в фундаменте и в приёмке.**
> Фиксируем не *результат* (HTML), а **(1) фундамент** (дизайн-токены + проверенная kit-библиотека) и **(2) приёмку** (авто-верификация и само-починка). Каталог-рендерер — страховочная сетка для аварийных случаев.

Референс — **base44**: модель пишет реальный код свободно, а надёжность даёт платформа (managed-примитивы «из коробки») + верификация. Надёжность перенесена с «модель не может ошибиться» на «фундамент крепкий + всё проверяется и чинится до показа».

## Целевой поток генерации

```
промпт
  → build_messages: методичка + ДИЗАЙН-ТОКЕНЫ (палитра+шрифты, выбраны с разбросом) + kit
  → СИЛЬНАЯ МОДЕЛЬ пишет вёрстку свободно → <file path="index.html">…</file>
  → ПРИЁМКА (sequential gate):
        1. структура — валидный HTML, kit подключён, ссылки живые, ровно один <h1>
        2. адаптив  — рендер 375/768/1440 → нет горизонтального скролла / наложений
        3. vision   — скриншот → модель: «сломано / генерик / красиво» + фидбек + score
        4. анти-генерик — похоже на прошлые / на «типовой AI-лендинг»? штраф
  → не прошло? точечная авто-починка с фидбеком (макс N=2–3)
  → исчерпано? АВАРИЙНЫЙ FALLBACK на каталог (трафареты живут только тут)
  → commit → snapshot
```

## Что переиспользуем (по итогам разведки кода)

~80% инфраструктуры уже есть — это не движок с нуля, а переключатель + промпт + слой приёмки.

| Уже есть (переиспользуем) | Доказательство |
|---|---|
| Freeform-путь `<file>HTML</file>` → commit | `messages.py:790` гейтит catalog только для premium+catalog; иначе `_extract_files_and_edits` (`messages.py:334-362`) → `extract_files` regex `<file path>` → `commit_files` (`messages.py:1342-1349`). Format-agnostic. |
| Playwright-рендер | `apps/api/src/omnia_api/workers/preview.py`: `_render_async` (async, можно `await` из пайплайна), грузит локальный `index.html`, `page.screenshot()` без `path=` отдаёт **bytes**. Сейчас 1 viewport `1280×800`, нет `page.evaluate`. |
| Retry-цикл (L6) | `messages.py` audit-retry loop — расширяем до полноценной приёмки. |
| `ui_audit` (10 проверок) + `link_validator` | детерминированная структурная база. |
| Vision-модели в gateway | `litellm_router.py`: `claude-sonnet-4-6`, `claude-opus-4-7`, `gpt-4.1→gpt-4o`, `gpt-5`, `gemini-2.5-pro/flash` — все vision-capable, LiteLLM сам форвардит `image_url` в Anthropic/OpenAI. |
| 54 палитры + методичка-промпт (2241 стр.) | `sections/palettes.py`, `services/prompt_builder.py`. |

**Новое пишем:** `vision_audit.py`, `acceptance.py`, `design_tokens.py`, `originality.py` + расширение `preview.py` (render-harness).

## План по спринтам

### Спринт 5 — Раскатка (размазана ~неделя)

Флаг `USE_FREEFORM_RENDER`: 10–20% трафика → метрики → 50% → 100%. Каталог-fallback включён весь период. Откат = выключить флаг (мгновенно).

## Флаги и настройки (итог)

```
USE_FREEFORM_RENDER       # bool — свободная вёрстка как основной путь (Спринт 3)
USE_ACCEPTANCE_GATE       # bool — включить приёмочный контроль (Спринт 2)
USE_VISION_AUDIT          # bool — vision-аудит внутри приёмки
ACCEPTANCE_MAX_RETRIES=2  # int  — максимум попыток авто-починки
ACCEPTANCE_MIN_SCORE=7    # int  — порог vision-score для прохождения
```

## Метрики раскатки (без них флаг не крутим)

- Средний vision-балл «красоты» ↑ · доля «генерик» ↓
- Разброс между сайтами ↑ (метрика непохожести из Спринта 4)
- Белые экраны / битая вёрстка → ~0 (держит приёмка)
- % генераций, прошедших приёмку с 1–2 попыток (контроль скорости)
- A/Б: freeform vs каталог по пользовательским оценкам и конверсии

## Риски и страховка

| Риск | Страховка |
|---|---|
| Свободная вёрстка ломается | Приёмка ловит до показа + каталог-fallback гарантирует валидный сайт |
| Дороже / медленнее (vision + retry) | Freeform только на сильных моделях; кэш; приёмка по ширинам параллельно |
| Vision-аудит субъективен | Чёткая рубрика + few-shot эталоны + связка с детерминированными проверками |
| Регресс качества/надёжности | Всё под флагами; метрики; мгновенный откат на каталог |

## Оценка

~6–7 недель суммарно. Первый видимый результат (приёмка + разброс палитр/шрифтов) — после Спринтов 1–2 (~3 недели), ещё до включения freeform.

## Связь с остальной системой

- **Не ломает контракт** (`docs/01-api-contract.md`): поток `POST /prompt` → WS-события `snapshot.created` тот же. Приёмка/починка — внутренние, между генерацией и commit.
- **Каталог/IR** (`sections/`) сохраняется целиком — переводится в статус fallback, не удаляется.
- **Fullstack** (`docs/07-v2-architecture.md`): тот же принцип «свобода + приёмка» применим к Next.js-генерации отдельной фазой (см. отдельные fullstack-болячки — `db:push`, миграции, build-проверка — в аудите; не входят в этот документ).
- **Координация:** gateway-правки (1.1) — зона Agent C, согласовать через inbox перед мержем; остальное — Agent B.
