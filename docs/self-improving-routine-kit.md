# Самоулучшающаяся рутина — KIT (можно скинуть другу)

Готовый набор + шаблоны, чтобы запустить у себя на компе **автономный цикл, который сам докручивает проект к твоей цели 24/7**. Ключевая идея: умнеет не модель (она фиксирована) — умнеет **АРТЕФАКТ**: план + его реализация. Каждый запуск перечитывает прошлое состояние из файлов и докручивает на один шаг.

> Это не «обучение модели». Это самоулучшающийся цикл над общими файлами. Рабочая, проверенная схема.

---

## Идея в одном абзаце

Две задачи по расписанию работают над общими файлами в одной папке:

- **ИСПОЛНИТЕЛЬ** (каждые ~10 мин): читает план + журнал прогресса → берёт следующий шаг → делает (код/тесты/деплой) → дописывает результат в журнал.
- **РЕФАКТОР** (каждые ~2 ч): читает план + прогресс → критикует и переписывает план амбициознее/чище → **но НЕ трогает зафиксированное ядро цели (NORTH STAR)**.

Чтобы это не «поехало» в сторону:
1. **NORTH STAR неприкосновенен** (рефактор его не правит).
2. **Версия плана + CHANGELOG** — видно что и почему поменялось, откатываемо через git.
3. **Renewing «dead-man» стоп** — рефактор каждый прогон катит горизонт на +72ч; пока приложение живо — цикл вечный; забросил >72ч — рутины сами затухают.

---

## Что нужно

- **Claude Code** (десктоп-приложение) с фичей Scheduled tasks (раздел «Scheduled» в сайдбаре).
- **Git-репозиторий** проекта (любой стек).
- Сформулированная **цель** — твоя путеводная звезда.
- (Опц.) доступ к серверу/проду для авто-доставки.

## Шаг 1 — создай файл плана `CONTINUOUS-PLAN.md` в корне репо

Это единственная общая память цикла. Скелет:

```markdown
# НЕПРЕРЫВНЫЙ ИНЖЕНЕР — план + протокол + прогресс + лог

> ВЕРСИЯ ПЛАНА: v1.0

## ★ NORTH STAR — ПУТЕВОДНАЯ ЗВЕЗДА (НЕПРИКОСНОВЕННО — рефактор НЕ ПРАВИТ это)
ЯДРО: <одно-два предложения: чего ты хочешь достичь, идеальный конечный результат>.
СТОЛПЫ (по ним меряется КАЖДАЯ задача):
1. <столп 1>
2. <столп 2>
3. <столп 3>
ИНВАРИАНТ: рефактор переписывает ТОЛЬКО forward-roadmap (§ROADMAP), не трогает NORTH STAR / правила / историю ЛОГА (append-only), bump версии + запись в CHANGELOG.

## 0c. HARD-STOP (renewing dead-man)
HARD-STOP = <дата+72ч> ← рефактор КАЖДЫЙ прогон катит на now+72ч.
Исполнитель: now >= HARD-STOP → сам себя выключает. Иначе работает.

## ПРАВИЛА (read first)
1. Не переходи к следующей задаче, пока текущая РЕАЛЬНО не работает (живой тест зелёный + логи чистые).
2. Полный цикл доставки: verify → commit → push → (деплой) → health-check.
3. Грязное дерево владельца: НИКОГДА stash/reset/checkout/`add -A`; только `git add <свои файлы>`.
4. Прод живой: health≠200 после деплоя → ОТКАТ на прошлый sha = приоритет №1.

## ROADMAP (forward — рефактор правит ТОЛЬКО это; `[ ]` pending, `[x]` done, `[~]` partial)
### Фаза 1 — <название>
- [ ] 1.1 <задача> + критерий приёмки (тест/E2E)
- [ ] 1.2 ...
### Фаза 2 — <название>
- [ ] 2.1 ...

## CHANGELOG ПЛАНА (append-only; каждая правка плана рефактором = запись)
- v1.0 — <дата> — initial.

## ПРОГРЕСС
<!-- исполнитель отмечает [x]/[~] выше + блок в ЛОГ -->

## ЛОГ ИТЕРАЦИЙ (append-only, новое сверху)
```

## Шаг 2 — рутина-ИСПОЛНИТЕЛЬ (prompt)

Создай scheduled task с этим промптом (подставь свои `<...>`):

```
Ты — EXECUTOR непрерывного самоулучшающегося цикла. ОДИН поток: одна задача за раз, до конца. Ты СВЕЖАЯ сессия без памяти; единственная память — <ПУТЬ_К_РЕПО>/CONTINUOUS-PLAN.md. Доступ санкционирован, не переспрашивай.

ПУТЕВОДНАЯ ЗВЕЗДА — читай §NORTH STAR в плане ПЕРВЫМ, она неприкосновенна.
РЕЖИМ: MAX EFFORT — думай тщательно (sequential-thinking), копай root cause до дна, исчерпывающе верифицируй, не срезай углы, [x] только при 100%.

0. СТОП-ГЕЙТ: прочитай HARD-STOP из §0c. now >= HARD-STOP → выключи задачу (update_scheduled_task enabled=false) → СТОП. Иначе дальше.
1. LOCK: файл <ПУТЬ_К_РЕПО>/.routine.lock. Если есть И моложе 25 мин → другой исполнитель идёт → СТОП молча. Иначе запиши «<UTC> <run-id>». Удали в конце.
2. КОНТЕКСТ: прочитай CONTINUOUS-PLAN.md целиком (NORTH STAR, ROADMAP, ПРОГРЕСС, хвост ЛОГА).
3. СИНХР: git fetch origin <ветка> → git merge --ff-only. Никогда stash/reset/checkout.
4. ВЫБОР: IN-FLIGHT → возобнови; иначе первый [ ] сверху по ROADMAP. Один связный слайс (1–3 файла).
5. ПЛАН задачи: sequential-thinking + context7 (актуальная дока по любой библиотеке).
6. RESOURCE-GUARD (БЕРЕЧЬ RAM): в начале тика убей осиротевшие automation-браузеры прошлых тиков (PowerShell: Get-Process | ? Path -match 'ms-playwright|puppeteer' и ProcessName chrome/chromium/headless_shell | Stop-Process -Force) — НИКОГДА рабочий браузер/приложение. Свободная RAM < ~3 ГБ → лёгкая задача, без тяжёлой генерации. Один браузер за раз, закрывай сразу.
7. РЕАЛИЗАЦИЯ — TDD-FIRST: баг-фикс → СНАЧАЛА failing-тест, потом фикс до зелёного. Новый код пишется ВМЕСТЕ с тестом (без покрытия = не сделано). Минимальный диф.
8. ЦИКЛ ДОСТАВКИ (строгий порядок):
   (a) ЛОКАЛЬНЫЙ ГЕЙТ — вся сюита затронутого: typecheck/lint/build + юнит/контракт-тесты (реальные команды твоего стека; не выдумывай несуществующие). Не пушить битое.
   (b) PUSH: commit (осмысленный) → git push origin <ветка>.
   (c) ДЕПЛОЙ затронутых рантайм-сервисов (если есть) → health-check 200, иначе ОТКАТ на прошлый sha.
   (d) ПОЛНЫЙ E2E как реальный юзер (headless-браузер): весь сценарий, desktop+mobile, скрин, логи (console+network+сервер). ПОСЛЕ теста ОБЯЗАТЕЛЬНО ЗАКРОЙ БРАУЗЕР.
   (e) ЗЕЛЁНО (все сюиты + E2E + 0 ошибок) → шаг 9. КРАСНО → НЕ переходи дальше: подними уровень логирования, ищи root cause (systematic-debugging), правь → снова (a)-(d), пока полностью не зелено.
9. ЖУРНАЛ + ПРИБОРКА: [x]/[~] в ПРОГРЕСС, блок в ЛОГ (сверху). Закрой браузер + добей осиротевшие automation-браузеры. Сними lock. Заверши тик ЧИСТО.

Ровно ОДНА связная задача за запуск, до полностью зелёного теста.
```

## Шаг 3 — рутина-РЕФАКТОР (prompt)

```
Ты — РЕФАКТОР-КАРЕТЕЙКЕР самоулучшающегося цикла. Раз в ~2ч: читаешь план, критикуешь, переписываешь ТОЛЬКО forward-roadmap амбициознее/чище — НЕ трогая зафиксированное ядро. Ты НЕ пишешь код, НЕ деплоишь, НЕ исполняешь задачи. Память — <ПУТЬ_К_РЕПО>/CONTINUOUS-PLAN.md.

0. ВРЕМЯ: запомни now.
1. КООРДИНАЦИЯ — ПРОХОДИШЬ В ЛЮБОМ СЛУЧАЕ. У тебя СВОЙ lock <ПУТЬ_К_РЕПО>/.refactor.lock — НЕ общий с исполнителем (иначе его вечная работа тебя блокирует). refactor.lock моложе 15 мин → другой рефактор идёт → СТОП. Иначе запиши и РАБОТАЙ даже если исполнитель активен (его lock игнорируй; он правит код+прогресс+лог, ты — только ROADMAP; параллельные правки разрулит git в шаге 6).
2. СИНХР: git fetch → git merge --ff-only.
3. ЧТЕНИЕ: NORTH STAR (read-only), ROADMAP, ПРОГРЕСС, хвост ЛОГА, CHANGELOG, git log.
4. КРИТИКА: целит ли roadmap в NORTH STAR? можно ли качественно амбициознее (не инкремент)? строги ли гейты/тесты? что устарело→убрать? новые идеи [ ]? (Можно мультиагентно через Workflow, если есть.)
5. ПЕРЕПИСЫВАНИЕ — ТОЛЬКО §ROADMAP (forward [ ]-задачи). НЕЛЬЗЯ: NORTH STAR, правила, история ЛОГА (append-only), уже-готовые [x]. Нечего улучшать → не выдумывай, только покати HARD-STOP.
6. HARD-STOP: обнови §0c на now+72ч.
7. ВЕРСИЯ + CHANGELOG: bump версии + запись «что изменил и ЗАЧЕМ».
8. ДОСТАВКА (docs-only, ПУШ-РЕТРАЙ): git add CONTINUOUS-PLAN.md → commit → push. Reject (исполнитель опередил) → git fetch → git merge origin/<ветка> (markdown авто-сольётся; конфликт → сохрани [x]/лог исполнителя, оставь свой roadmap-рерайт) → push. До 3 попыток. Никогда stash/reset/checkout. Сними refactor.lock.
```

## Шаг 4 — зарегистрируй обе как Scheduled tasks

В Claude Code (десктоп) попроси:
> «Создай scheduled task `<id>` с cron `<...>` и этим промптом: …»

Или открой панель **Scheduled** и добавь вручную. Параметры:

| Задача | cron | модель | effort | permission |
|---|---|---|---|---|
| исполнитель | `*/10 * * * *` | посильнее (Opus) | max | bypassPermissions |
| рефактор | `17 */2 * * *` (нечётная минута) | посильнее | max | bypassPermissions |

> Эффорт (`max`) и модель выставляются в диалоге задачи / Run-now. `ultracode` (мультиагентная оркестрация рефактора) включается ключевым словом `ultracode` прямо в промпте задачи.

## ⚠️ Чтобы заработало ИМЕННО как у автора — адаптируй под себя (6 пунктов)

Движок (2 рутины + координация + safety rails) переносится 1:1. Но это **шаблон** — заполни своё, иначе не поедет:

1. **Пути** — замени все `<ПУТЬ_К_РЕПО>` на абсолютный путь своего репо.
2. **Git-аутентификация** — рутины пушат сами. Настрой доступ заранее (`gh auth login`, git credential manager или свой PAT). В шаблоне НЕТ чужих секретов — подставь свои.
3. **NORTH STAR + ROADMAP** — впиши СВОЮ цель и первые `[ ]`-задачи. Без этого исполнителю нечего брать.
4. **Реальные команды стека** — в локальном гейте (шаг 8a) укажи СВОИ: `npm test` / `pytest` / `go test` / `cargo test` + typecheck/lint/build. Не оставляй выдуманные.
5. **Деплой (шаг 8c)** — нет прода → УБЕРИ этот под-шаг (оставь push). Есть → впиши свои deploy-команды + health-check.
6. **Настройки Claude Code** — в диалоге задачи: модель посильнее, effort `max`, `bypassPermissions` (иначе ночью застрянет на пермишене). MCP-серверы, что юзают твои задачи (context7 / sequential-thinking и т.п.) — поставь заранее.

## Координация — один комп vs ДВА компа (общий проект)

**Один комп (соло):**
- Исполнитель — общий lock `.routine.lock` (один исполнитель за раз, ноль наложений).
- Рефактор — СВОЙ lock `.refactor.lock`, НЕ общий (иначе вечная работа исполнителя его блокирует). Проходит в любом случае; конкуренцию за файл плана разруливает git (push-retry + merge: исполнитель правит прогресс/лог/чекбоксы, рефактор — forward-roadmap, обычно разные строки).

**ДВА компа = один проект, два со-владельца (КРИТИЧНО!):**
Локальный lock-файл **НЕ** сериализует между машинами. Если оба запустят исполнителя наивно → двойной деплой на один прод, конфликтные пуши, дубль тест-окружений. Нужен ОБЩИЙ мьютекс:
- **Общий замок на shared-сервере (рекомендую)** — atomic `mkdir` на сервере, к которому SSH у обоих: `ssh server 'L=~/.exec-lock; find "$L" -maxdepth 0 -mmin +25 -exec rm -rf {} \; 2>/dev/null; mkdir "$L" 2>/dev/null && echo OK || echo BUSY'` → `OK` работаешь, `BUSY` стоп; в конце `ssh server 'rm -rf ~/.exec-lock'`. Побеждает ровно ОДИН между ЛЮБЫМИ машинами; протухший >25мин авто-снос. Без шума в git-истории.
- **Git-замок** (если общего сервера нет) — lock-файл коммитится+пушится в репо; push-reject = мьютекс. Минус: шум в истории.
- Перед работой ВСЕГДА `git pull --ff-only`; деплой делает ТОЛЬКО держатель замка.

**Роли на двоих (просто, без коллизий):**
- Исполнитель — на ОБОИХ компах, сериализован общим замком → 24/7 редундантность (один закрыл ноут — второй держит цикл). Один билд за раз (не параллельно, но непрерывно).
- Рефактор — на ОДНОМ компе достаточно (план — один каретейкер). На обоих → свой общий замок рефактора.
- Нужна ИСТИННАЯ параллельность (оба билдят разом) → раздели ЗОНЫ: один комп берёт фронт, другой бэк (разные файлы/сервисы) — тогда не дерутся. Помечай задачи в плане зоной.

## 👥 Co-owner setup — второй владелец на своём компе (ТОТ ЖЕ репо)

Напарник поднимает ТУ ЖЕ рутину на ОБЩЕМ проекте:
1. Клонировать **НАШ** репо (тот же remote), НЕ заводить новый.
2. Свой git push-доступ на тот же репо (свой PAT / `gh auth login`).
3. SSH-алиас к ОБЩЕМУ серверу (тот же VPS) — для общего замка + деплоя.
4. Claude Code: те же 2 scheduled-задачи (промпты Шаг 2/3), effort=max, модель посильнее, bypassPermissions. Lock — через ОБЩИЙ VPS-замок (см. выше), НЕ локальный файл.
5. (реком.) у него — только ИСПОЛНИТЕЛЬ; рефактор оставь на одном компе. Либо договоритесь о зонах.
6. Оставить приложение Claude открытым.

Готово → две машины ведут ОДИН проект к одной звезде, без коллизий.

## Гигиена RAM / процессов (важно!)

Каждый тик = новая сессия Claude Code, которая поднимает ВСЕ настроенные MCP-серверы. Много параллельных лупов × много MCP = десятки процессов и забитая RAM.

- Закрывай браузер после КАЖДОГО теста; в начале/конце тика добивай осиротевшие automation-браузеры (путь под `ms-playwright`/`puppeteer`), не трогая рабочий браузер.
- Один тяжёлый процесс за раз. Свободной RAM < ~3 ГБ → лёгкая задача, без тяжёлой генерации.
- **Не держи много лупов одновременно.** Главный пожиратель RAM — параллельные сессии и их MCP-серверы.
- Оставь только нужные MCP-серверы (лишние — отключи в конфиге): меньше серверов = меньше процессов на сессию.
- Заверши тик ЧИСТО (lock снят, браузер закрыт) → дерево процессов сессии умирает само.

---

## Safety rails (почему это не «поедет»)

1. **NORTH STAR неприкосновенен** — рефактор его не правит, цель не дрейфует.
2. **Версия + CHANGELOG** — каждое изменение плана видно и откатываемо через git.
3. **Renewing dead-man HARD-STOP** — живёт пока приложение тикает; заброшено → затухает.
4. **Лог append-only** — история не переписывается.
5. **Локальный гейт перед пушем** + **откат прода при health≠200** — сломанное не доезжает.

## Как смотреть прогресс

- `git log --oneline` в репо — поток коммитов от рутин.
- `CONTINUOUS-PLAN.md` → §ПРОГРЕСС и §ЛОГ — что сделано.
- (Опц.) отдельная задача-«утренний отчёт» в 08:00 — соберёт простым языком, что сделано за ночь.

> Оставь ноут и приложение Claude включёнными — scheduled-задачи идут только пока приложение открыто (закрыл — добьются на следующем запуске).
