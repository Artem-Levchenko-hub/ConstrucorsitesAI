# 04. Свод правил генерации — как AI пишет сайты в Omnia.AI

> **Цель документа.** Зафиксировать единый стандарт того, что Omnia.AI выдаёт
> пользователю по промпту. Принцип один: **«готовое решение, а не страничка»** —
> один промпт обязан давать законченный продукт enterprise-уровня.
>
> **Операционный источник правил** — система-промпт в
> [`apps/api/src/omnia_api/services/prompt_builder.py`](../apps/api/src/omnia_api/services/prompt_builder.py).
> Этот документ — человекочитаемая версия с обоснованием. Меняешь правила —
> синхронизируй оба.

## 1. Static-first: почему статика по умолчанию

Превью обязано показывать сайт **без исключений и мгновенно**. Этого даёт только
статика:

- **Статический сайт** (`index.html` + Tailwind CDN + vanilla JS) рендерится
  напрямую из git через `/p/<slug>` за миллисекунды. Контейнер не нужен.
- **Next.js/full-stack** рендерится только в живом dev-контейнере (оркестратор +
  Docker + nginx), с холодным стартом 30–60 c. Пока контейнер не поднят —
  показать нечего. Это и есть источник «белого экрана на TypeScript».

Поэтому **режим определяется типом проекта** (`projects.template`), а не желанием
модели писать модный стек:

| Тип проекта (`template`) | Режим | Чем рендерится превью |
|---|---|---|
| `blank`, `landing`, `portfolio`, `blog` | **Статика** | `/p/<slug>` (git → файл) |
| `fullstack` | **Next.js + Postgres + Drizzle** | dev-контейнер оркестратора |

**Следствие для модели:** в статическом проекте НИКОГДА не генерируй `.tsx`,
`package.json` и прочий Next.js — его нечем показать. Если задача реально требует
сервера (личный кабинет, оплата, хранение заявок в БД) — сделай лучший
статический вариант и одной строкой предложи создать full-stack проект.

## 2. Стандарт качества (применяется к обоим режимам)

### 2.1 Полнота
Лендинг — это **7–9 осмысленных секций**, а не один экран: sticky-шапка → hero →
блок доверия → ценность/возможности → как это работает → отзывы → цены → FAQ →
финальный CTA → футер. Набор адаптируется под продукт (портфолио, блог, меню,
мероприятие), но страница всегда цельная и «прокручиваемая».

### 2.2 Контент
Настоящий и конкретный, по-русски, в тоне бренда: реальные офферы, цифры, цены в
₽, имена, города. **Запрещено:** lorem ipsum, «Заголовок 1», «ваш текст здесь»,
повтор одного абзаца, пустые `<a href="#">`. Нет данных — придумай правдоподобные.

### 2.3 Дизайн-система (Refactoring UI)
- Одна палитра: фирменный/акцентный цвет + нейтральная шкала (slate/zinc/stone,
  **не** чистый `#000` на `#fff`) + семантика. Контраст текста ≥ **4.5:1**.
- Максимум 2 шрифта (заголовочный + текстовый), Google Fonts с `display=swap`.
  База 16px, line-height 1.5–1.7, заголовки — крупные, `tracking-tight`.
- Типографическая шкала (14/16/18/20/24/30/36/48/60), не случайные размеры.
- Ритм отступов кратен **4/8**. Секции просторные (`py-16…py-28`), контейнер
  `max-w-6xl/7xl mx-auto px-6`. Глубина — слоями (фон → карточка → мягкая тень),
  а не рамками везде. Консистентные скругления.

### 2.4 Иерархия и полировка
Одна главная CTA на экран, вторичные приглушены. У всех кликабельных элементов —
`hover`/`focus` состояния и `transition` 150–300 мс. Иконки — **инлайновый SVG**
(Heroicons/Lucide, stroke 1.5–2), **никогда не эмодзи**. Микро-анимации появления
секций — ненавязчиво и с уважением к `prefers-reduced-motion`.

### 2.5 Адаптив (mobile-first)
Корректно на 375 / 768 / 1024 / 1440. Без горизонтального скролла. На мобиле —
рабочее бургер-меню. Сетки перестраиваются (`grid-cols-1 → md:grid-cols-3`).

### 2.6 Доступность
Семантика (`header/nav/main/section/footer`), заголовки `h1→h6` без пропусков,
ровно один `<h1>`. `alt` у картинок, `aria-label` у иконочных кнопок, видимый
focus-ring. Смысл — не только цветом.

### 2.7 SEO и мета
Уникальный `<title>`, `<meta name="description">`, `<html lang="ru">`, Open Graph
(`og:title/description/type/image`), favicon. Осмысленная иерархия заголовков.

### 2.8 Интерактив
Рабочий, не декоративный: мобильное меню, плавный скролл по якорям, аккордеон
FAQ, клиентская валидация формы, кнопка «наверх». Vanilla JS.

### 2.9 Цельность
Один визуальный язык на весь сайт: единые кнопки, карточки, ритм секций.

## 3. Статический стек (режим по умолчанию)

- Чистый HTML5 + CSS + vanilla JS, без сборщиков и npm.
- Tailwind через CDN; кастомизация — инлайн `tailwind.config = {…}` перед
  использованием. Шрифты — Google Fonts через `<link>`.
- **Главная — всегда `index.html` в корне репозитория.** Это то, что рендерит
  превью; без него сайт не покажется. В любом ответе, меняющем вёрстку, корневой
  `index.html` обязан присутствовать.
- Доп. страницы — отдельные `.html` в корне, относительные ссылки.
- Картинки — `picsum.photos` / `placehold.co` с `alt` и заданными размерами,
  либо инлайновый SVG / CSS-градиент.
- Формы — `mailto:` или внешний приёмник (formspree) + клиентская валидация.

## 4. Full-stack стек (только для `fullstack`-проектов)

Next.js 15 App Router + React 19 + TypeScript + Tailwind v4 + Drizzle ORM. Полный
свод конвенций (импорты Drizzle, server actions, что нельзя трогать) — в
[`apps/orchestrator/templates/nextjs-postgres-drizzle/SYSTEM_PROMPT.md`](../apps/orchestrator/templates/nextjs-postgres-drizzle/SYSTEM_PROMPT.md)
и в `_FULLSTACK_STACK` промпта. Стандарт качества из §2 применяется и здесь.

## 5. Анти-примеры: «страничка» vs «готовое решение»

| ❌ Страничка (так НЕ надо) | ✅ Готовое решение |
|---|---|
| Hero + 3 пустые карточки + «Цены» без цен | 8 секций с реальным контентом и ценами в ₽ |
| «Заголовок, бьющий в боль клиента» | «Запишитесь на стрижку за 60 секунд — мастера с опытом 8+ лет» |
| `#000` на `#fff`, один размер шрифта | палитра slate + акцент, типошкала, ритм 4/8 |
| Эмодзи вместо иконок | инлайновый SVG |
| Нет адаптива, нет focus-состояний | mobile-first, бургер-меню, hover/focus |
| `.tsx` в статическом проекте → белый экран | корневой `index.html`, рендерится сразу |

## 6. Где это включено в коде

- **Промпт:** `prompt_builder.py` → `_IDENTITY` + `_QUALITY_BAR` + (`_STATIC_STACK`
  \| `_FULLSTACK_STACK`) + `_RESPONSE`; режим выбирает `build_system_prompt(template)`.
- **Передача типа проекта:** `routers/messages.py` читает `project.template` и
  отдаёт его в `build_messages(...)`.
- **Надёжность превью:** `routers/public.py` никогда не отдаёт пустой экран (см.
  [`docs/07-v2-architecture.md`](07-v2-architecture.md) про full-stack runtime).
