# Phase L — отчёт о работе (сессия 2026-05-27 → 28)

## Зачем всё это делалось

Цель сессии: научить Omnia.AI генерировать сайты дизайн-уровня awwwards
на сильных моделях (Opus / Sonnet / GPT-5) — и при этом тратить меньше
токенов, чем раньше. Плюс по ходу закрыли несколько проблем с
надёжностью (зависающие запросы) и качеством цвета.

Главная идея: раньше модель получала системный промпт на ~14 000 токенов
сплошной прозы и писала сайт целиком в HTML. Это давало нестабильный
результат — ключевые правила «тонули» в длинном тексте, модель
импровизировала палитру и вёрстку. Мы развернули модель на работу с
**каталогом готовых секций** и **JSON-описанием страницы** вместо
свободного HTML: модель выбирает из готовых блоков, а финальный HTML
собирает детерминированный рендерер. Это даёт стабильный «потолок
качества» и резко экономит токены.

---

## Что конкретно сделали

### Архитектура генерации (Phase L1–L3)

- **Каталог секций.** 16 готовых вариантов блоков (header, hero ×3,
  features ×2, pricing ×2, cta ×2, testimonials, faq, footer, contact,
  stats, about) с Pydantic-схемой и Jinja-шаблонами. Модель выбирает
  `hero.v3`, `pricing.v1` и т.д., а не пишет вёрстку руками.
- **JSON IR (Intermediate Representation).** Модель возвращает один
  JSON-объект (`PageIR`): метаданные, тема, список секций. Рендерер
  превращает его в HTML локально. Невалидный JSON → откат на старый
  путь, чтобы пользователь всё равно получил сайт.
- **Фиче-флаг `USE_SECTION_CATALOG`** — включает новый путь без риска
  для старого. На проде включён только для премиум-моделей.

### Экономия токенов и качество промпта (Phase L4)

- **Lean-промпт** вынесен в отдельный модуль. Размер системного блока
  упал с ~54 000 символов до ~2 800 (примерно в 19 раз).
- Использованы приёмы из руководства Anthropic по prompt-engineering:
  XML-теги для структуры, закрытый список «вайбов» вместо описаний,
  палитра вынесена в «хвост» промпта (модель лучше помнит начало и
  конец).
- 8 «вайбов» дизайна как enum: swiss-minimal, apple-tech, linear-dark,
  fintech-trust, editorial-luxury, brutalist, glassmorphism, y2k-neo.

### Подсказки-референсы (Phase L5)

- **RAG по awwwards-корпусу.** К каждому запросу подмешивается один
  релевантный референс из готовой библиотеки (палитра, шрифты, приёмы
  движения, «почему это работает»). Без тяжёлых эмбеддингов — быстрый
  поиск по ключевым словам, корпус маленький.

### Самоконтроль качества (Phase L6)

- **Авто-ретрай по аудиту.** Если объективная проверка дизайна
  (`ui_audit`) даёт меньше 7/10 — система один раз переспрашивает модель
  с конкретным списком нарушений и просит исправить. Без бесконечных
  циклов — максимум одна попытка.

### Двухпроходный режим для премиума (Phase L7)

- **Director → Polish.** Опциональный режим (флаг `USE_DIRECTOR_POLISH`):
  первый проход выбирает структуру, второй наполняет реальным контентом.
  Включается отдельно, по умолчанию выключен.

### Умные дефолты (Phase L8)

- Постобработка JSON перед рендером закрывает «дыры», которые модель
  иногда оставляет: пустые CTA-ссылки ведут на блок контактов,
  выделяется «популярный» тариф, проставляется копирайт с годом, фавикон
  по индустрии, согласуется тёмная тема, дедуплицируются якоря.

### Анимации (Phase L9)

- **В сгенерированные сайты** (`omnia-kit.js`) добавлены 4 эффекта на
  нативном Web Animations API (без внешних зависимостей):
  - `anime-hero-reveal` — заголовок проявляется по словам/буквам;
  - `data-anime="fade-up"` — секции всплывают при скролле;
  - `data-anime-counter` — счётчики крутятся от 0 до значения;
  - `data-anime-magnetic` — кнопки притягиваются к курсору.
  Все уважают `prefers-reduced-motion`.
- **В UI продукта** добавлен пакет `animejs` и React-компонент
  `AnimeReveal` (тот же эффект, та же кривая анимации — для
  единообразия).

### Система цвета (Phase L10)

По мотивам книги Йозефа Альберса «Interaction of Color»:

- **66 готовых палитр** по 9 вайбам, светлые и тёмные. У каждой —
  полный набор: основной/акцентный цвет, фон, поверхность, текст,
  приглушённый текст, границы, плюс семантика (успех/предупреждение/ошибка).
- **Контраст по WCAG AA** проверяется на этапе сборки — текст к фону
  ≥ 4.5:1, приглушённый текст ≥ 3:1. Все 66 палитр проходят проверку
  автотестами.
- **Правило Альберса**: запрещена комбинация чистый `#000` на чистом
  `#FFF` (глаз «дрожит» на резком контрасте) — нет ни в одной палитре.
- Палитра подбирается автоматически: индустрия → вайб → палитра, с
  учётом светлой/тёмной темы. Пользователь больше не получит
  «нечитаемые надписи» или случайный подбор цветов.

### Надёжность запросов (отдельный блок фиксов)

Была проблема: пользователь пишет запрос, видит «AI читает контекст» и
может ждать вечно, если что-то ломалось. Закрыли на обоих концах:

- **Бэкенд:** если фоновая задача генерации падает или отменяется — она
  гарантированно публикует событие ошибки и финализирует сообщение в
  БД. Пользователь всегда видит явную ошибку, а не вечный спиннер.
- **Фронтенд:** POST-запрос на генерацию получил таймаут 30 секунд +
  обработку всех ошибок (сеть, 4xx, 5xx) с показом тоста. Плюс
  watchdog на 90 секунд: если от модели нет ни одного события —
  пользователь видит понятное сообщение.
- **Маршрутизация по тарифу:** дешёвые модели (Haiku/Nano) остались на
  старом multipass-пути, каталог только для премиума — это убрало
  конфликт, из-за которого дешёвые модели «зависали» на минуту.

## Покрытие плана awwwards (15 пунктов)

Все 15 пунктов исходного плана реализованы:

| # | Пункт | Где |
|---|---|---|
| 1 | Pre-bake design tokens | Theme + CSS-переменные |
| 2 | Каталог компонентов | 16 вариантов секций |
| 3 | JSON IR вместо HTML | PageIR |
| 4 | Style packs (вайбы) | 8 вайбов как enum |
| 5 | Industry templates | preset_classifier + маппинг на вайб |
| 6 | Двухпроходный дешёвый пайплайн | multipass (Haiku/Nano) |
| 7 | RAG из awwwards | top-1 референс |
| 8 | Системный промпт 14K → 4K | ~2,8K |
| 9 | Жёсткие ограничения | схема + ретрай |
| 10 | Визуальный декор | visual_enricher |
| 11 | Картинки вне LLM | image_resolver |
| 12 | Пресеты анимаций | omnia-kit + Phase L9 |
| 13 | Golden-гейт с ретраем | score < 7 → ретрай |
| 14 | Фиксы на этапе компиляции | Jinja autoescape |
| 15 | Движок умных дефолтов | Phase L8 |

Плюс сверх плана: Director→Polish, 66 палитр с WCAG-валидацией,
animejs-эффекты, гарантии надёжности запросов.

---

## Цифры

- **Токены (премиум, один запрос):** системный промпт −95% (54K → 2,8K
  символов), вывод −60% (HTML → JSON IR). Суммарно примерно −73%.
- **Тесты:** 160 автотестов, все проходят (было 83 в начале сессии).
- **Палитры:** 66 штук, 100% проходят WCAG AA.

## Состояние на проде

- Ветка `main`, последний коммит развёрнут на VPS
  (`constructor.lead-generator.ru`).
- Каталог-режим включён (`USE_SECTION_CATALOG=true`) для премиум-моделей.
- Director→Polish выключен (`USE_DIRECTOR_POLISH=false`) — включается при
  необходимости.
- Дешёвые модели (Haiku, Nano) работают по старому пути без изменений.

## Как откатить

Если что-то пойдёт не так — выключить каталог-режим:

```bash
ssh i48ptgvnis@170.168.72.200
cd /opt/omnia/apps/llm-gateway/deploy/full
sed -i 's/^USE_SECTION_CATALOG=true/USE_SECTION_CATALOG=false/' .env
docker compose -p full restart api worker
```

Старый свободный путь генерации вернётся через ~10 секунд.
