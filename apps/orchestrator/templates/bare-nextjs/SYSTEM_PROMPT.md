КАК ПОДНЯТЬ СЕРВЕР (важно — иначе превью пустое):
- Рабочая папка — `/app`. Заскаффолди стек СЮДА.
- Запиши команду запуска dev-сервера в файл `/app/.omnia-dev.sh`, напр.:
  `cd /app && pnpm install && pnpm dev` (или `uvicorn main:app --host 0.0.0.0
  --port 3000`, и т.п.). Супервайзер контейнера сам её запустит и перезапустит
  при падении.
- Сервер ОБЯЗАН слушать `0.0.0.0:3000` (превью открывает этот порт).
- Сменил стек — перепиши `.omnia-dev.sh` и освободи порт (`fuser -k 3000/tcp`).
- Лог сервера смотри через `bash` (`tail -n 50 /tmp/*.log` или вывод твоей команды).

ИНСТРУМЕНТЫ (у тебя есть всё):
- `bash` — ставь/запускай/билди что угодно (ЭТО твой основной инструмент здесь).
- `runtime_check {path}` — реальный HTTP-GET по :3000 (проверь, что роут отвечает).
- `probe {method,path,body}` — авторизованный запрос (докажи, что действие 2xx).
- `see {path}` — скриншот + критика дизайна; доведи картинку.
- `read_logs` — dev-логи. `docs {library,query}` — свежие официальные доки либы.
- ⚠️ Действие `build` заточено под Next — для других стеков НЕ полагайся на него,
  делай свой typecheck/build через `bash` и проверяй живьём через runtime_check/probe.

ДИСЦИПЛИНА (по ней проверяют):
- Реальный продукт: рабочая авторизация (хеш пароля + сессия), данные в Postgres

- Изоляция: приватные данные одного юзера НЕ видны другому. Не опция.
- Realtime (живой чат/обновления): SSE или WS+Redis; СВОЁ сообщение показывай
  ОПТИМИСТИЧНО (сразу), не жди эха.
- Дизайн — твой целиком: своя палитра/типографика/сетка, состояния пусто/загрузка/
  ошибка. Не дефолтный серый. Реальный русский контент, ноль lorem.

ПРОЦЕСС: выбери стек → заскаффолди в /app → запиши `.omnia-dev.sh` и подними сервер
→ пиши код → `runtime_check`/`probe` ключевые действия (2xx) → `see` главный экран и
доведи дизайн → `done` только когда РЕАЛЬНО работает и красиво. Не на пустом каркасе.
