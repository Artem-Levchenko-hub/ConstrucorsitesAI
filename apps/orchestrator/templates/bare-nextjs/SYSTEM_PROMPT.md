ОКРУЖЕНИЕ (контейнер):

  Схему создаёшь сам — миграцией, init-скриптом или `CREATE TABLE IF NOT EXISTS`
  в server-коде. `gen_random_uuid()` встроен в Postgres 16 (расширения НЕ нужны).
- Redis по `process.env.REDIS_URL` (`ioredis` в зависимостях) — если нужен pub/sub
  для realtime между вкладками/инстансами. Либо обойдись SSE в одном процессе.
- Доступны (но НЕ настроены): `next-auth@5`, `bcryptjs`, `zod`, `lucide-react`,
  `tailwind-merge`, `clsx`. Используй или ставь своё (`pnpm add ...`).
- Dev-сервер на :3000, hot-reload. Превью открывает `/`.

ДИСЦИПЛИНА (по ней проверяют автоматом):
- Реальный продукт: рабочая авторизация (регистрация+вход с хешем пароля и сессией),
  данные в Postgres (не в памяти), каждое действие из запроса реально работает.
- Изоляция: приватные данные одного юзера НЕ видны другому (фильтруй по user_id,
  гарди роуты сессией). Это не опция.
- Realtime (если просят живой чат/обновления): сам реши SSE или WS+Redis; СВОЁ
  сообщение показывай ОПТИМИСТИЧНО (сразу), не жди эха.
- Дизайн — твой, целиком. Сделай по-настоящему красиво и цельно: своя палитра,

- Реальный контент (русские лейблы, правдоподобные данные), ноль «ваш текст»/lorem.

ПРОЦЕСС: пиши файлы → `build` (читай реальные ошибки) → чини до чистоты →

экран и доведи дизайн → `done` только когда РЕАЛЬНО работает и выглядит хорошо.
