# Skill: a11y — pass the accessibility floor

Read before any page/component with images, interactive elements, forms, or
color. The **perf-a11y gate runs axe-core on the live render and BLOCKS on any
serious/critical violation** (missing alt, unnamed controls, unlabelled inputs,
missing lang/title). Build it right the first time.

## The rules that actually fail the gate
- **Every image has alt** (WCAG 1.1.1). Informative: `alt="описание"`. Decorative:
  `alt=""` (still required). Use `next/image` `<Image>` with `alt` — it also helps
  the perf budget.
- **Every control has an accessible name.** A `<button>`/`<a>` must have visible
  text, or an `aria-label` if icon-only: `<Button aria-label="Удалить"><Trash/></Button>`.
- **Every input has a label.** Associate a `<label htmlFor="email">` with
  `<input id="email">`, or use the kit's form field which wires it for you.
- **Interactive = a real interactive element.** Don't put `onClick` on a `<div>`/
  `<span>`. Use `<button>` (keyboard + screen-reader work for free). Positive
  `tabIndex` is banned — use `0` or `-1`.
- **One `<h1>` per page**, headings in order (don't skip levels for styling).
- **`<html lang>` and a page `<title>`** — the layout/metadata already set these;
  don't remove them.

## Get most of it for free
- Compose the kit: `@/components/ui/*` (shadcn) ships focus rings, ARIA, and
  keyboard behaviour; `@/components/omnia/*` (AppShell, CrudResource, forms) are
  accessible by construction. Don't hand-roll a dropdown/dialog/table.
- Lucide icons inside a labelled control; never an emoji as the only content.
- Color contrast is owned by a separate visual gate — use design tokens
  (`text-foreground` on `bg-background`), never low-contrast custom hex.

Quick self-check before `done`: every `<img>`/`<Image>` has `alt`; every
icon-only button has `aria-label`; every input is labelled; no `onClick` on a
`div`; one `<h1>`.
